Porque testes unitários são importantes?

  • Eles dão um feedback muito rápido sobre o código
  • Apontam onde o seu seu código está quebrando e/ou efeitos colaterais da implementação
  • Dão mais confiança em alterar o código
  • Podem garantir a manutenção da "testabilidade" do código
  • Ajudam na melhor compreensão do funcionamento do sistema

Como identificar um "mal cheiro" (bad smell)

  1. Código duplicado
  2. Métodos longos (Certeza que possuem muitas responsabilidades)
  3. Alto acoplamento
  4. Encadeamento de mensagens
  5. Classes de dados (dicionáriooo?!?! Te amo!! ;* )
  6. Códigos complexos no geral
